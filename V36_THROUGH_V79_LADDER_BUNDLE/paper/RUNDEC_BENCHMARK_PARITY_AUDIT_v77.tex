
\section{v77: RunDec Benchmark Parity Audit}
\label{sec:v77-rundec-benchmark-parity}

This module tests whether the vendored fixed-flavour four-loop evaluator of v76 is already equivalent to a full RunDec/CRunDec evaluator.  The answer is no: v76 is a useful source-clean fixed-$n_f$ subset, but it is not full RunDec parity.

\subsection{Benchmark sources}
The RunDec/CRunDec v3 useful-examples ledger gives, among its standard inputs, $m_c(m_c)=1.279\,\mathrm{GeV}$ and $m_c(3\,\mathrm{GeV})=0.986\,\mathrm{GeV}$ in the full RunDec context.  The CRunDec example ledger also gives the inverse-style bottom benchmark $m_b(10\,\mathrm{GeV})=3.610\,\mathrm{GeV}\mapsto m_b(m_b)=4.163\,\mathrm{GeV}$ using $\alpha_s^{(5)}(M_Z)=0.1189$.

\subsection{Fixed-$n_f$ parity audit}
The v76 evaluator uses fixed active flavour number and a local input $\alpha_s(\mu_0)$; it does not run from $M_Z$, perform decoupling, or select matching thresholds.  Running the published benchmark shapes through the v76 subset gives:
\[
  m_c(3\,\mathrm{GeV})_{\rm v76\ subset}=0.998662084561\,\mathrm{GeV}
\]
from $m_c(m_c)=1.279\,\mathrm{GeV}$ and $\alpha_s(m_c)=0.38$, compared with the RunDec v3 example value
\[
  0.986\,\mathrm{GeV}.
\]
For bottom, the reverse audit gives
\[
  m_b(10\,\mathrm{GeV})_{\rm v76\ subset}=3.623284185245\,\mathrm{GeV}
\]
from $m_b(m_b)=4.163\,\mathrm{GeV}$ and $\alpha_s(m_b)=0.223$, compared with the CRunDec example target
\[
  3.610\,\mathrm{GeV}.
\]

\subsection{Conclusion}
The benchmark mismatch is not an APF failure.  It identifies the remaining external-evaluator obligation:
\[
\boxed{
\text{full RunDec parity requires either a validated RunDec/CRunDec adapter or an implemented }\alpha_s(M_Z)\text{ + threshold/matching evaluator.}
}
\]
Therefore v77 upgrades the status to
\[
\boxed{
 c,b:[P_{\rm fixed\text{-}n_f\ 4loop\ evaluator}]
 +[P_{\rm benchmark\ audit}]
 /[P_{\rm full\ RunDec\ parity\ pending\ adapter/matching}].
}
\]
It does not claim full RunDec parity, threshold matching, or physical-final charm/bottom masses.
