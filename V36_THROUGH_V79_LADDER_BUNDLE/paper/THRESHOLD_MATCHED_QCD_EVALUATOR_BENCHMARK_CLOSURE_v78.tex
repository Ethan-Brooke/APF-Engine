
\section{v78: Threshold-Matched QCD Evaluator Benchmark Closure}
\label{sec:v78-threshold-matched-qcd}

This module repairs the v77 benchmark mismatch by adding the missing external input path: the evaluator no longer starts from a rough local value of $\alpha_s(m_q)$, but from an external $\alpha_s^{(5)}(M_Z)$ benchmark ledger and propagates $\alpha_s$ through a threshold-matched active-flavour scaffold.

\subsection{Protocol}
The v78 protocol uses
\[
  \alpha_s^{(5)}(M_Z)=0.1189,\qquad M_Z=91.1876\,{\rm GeV},
\]
then runs with $n_f=5$ down to the bottom threshold and $n_f=4$ below it.  At this stage the threshold is continuous in $\alpha_s$: this is a RunDec-compatible scaffold and benchmark-closure layer, not a claim of complete RunDec parity with all higher-loop decoupling constants.

\subsection{Published benchmark closure}
For the charm RunDec benchmark,
\[
  m_c(m_c)=1.279\,{\rm GeV}\longrightarrow m_c(3\,{\rm GeV}),
\]
v78 obtains
\[
  m_c(3\,{\rm GeV})_{v78}=0.986551889088\,{\rm GeV},
\]
compared with the published RunDec example value
\[
  0.986\,{\rm GeV}.
\]
The residual is
\[
  +0.551889088\,{\rm MeV}.
\]
For the bottom CRunDec benchmark,
\[
  m_b(m_b)=4.163\,{\rm GeV}\longrightarrow m_b(10\,{\rm GeV}),
\]
v78 obtains
\[
  m_b(10\,{\rm GeV})_{v78}=3.610111846228\,{\rm GeV},
\]
compared with the benchmark value
\[
  3.610\,{\rm GeV}.
\]
The residual is
\[
  +0.111846228\,{\rm MeV}.
\]
Both are below the declared $1\,{\rm MeV}$ benchmark tolerance.

\subsection{APF charm/bottom transport under the same protocol}
Applying the same external $\alpha_s(M_Z)$ threshold protocol to the APF self-scale inputs gives
\[
  m_c^{\rm APF}(3\,{\rm GeV})=0.979226596181\,{\rm GeV},
\]
and
\[
  m_b^{\rm APF}(10\,{\rm GeV})=3.625054969391\,{\rm GeV}.
\]
These are external-evaluator outputs, not new APF derivations.
